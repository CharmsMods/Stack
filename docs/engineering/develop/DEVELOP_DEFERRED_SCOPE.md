# Develop Deferred Scope

Last updated: 2026-06-22

This file prevents accidental overclaiming and reimplementation. Deferred items may be documented or prepared for, but they must not be claimed implemented until the owning guide/pass actually implements them and updates the tracker.

Status meanings:

- `Not Started`: no implementation should be assumed to exist, even if a planning guide describes the target behavior.
- `Partial`: some code or diagnostics exist, but the owning guide still has explicit unmet requirements in `DEVELOP_IMPLEMENTATION_TRACKER.md`.
- `Deferred`: intentionally out of scope until a future explicit pass.
- A deferred feature can move to `Partial` or `Complete` only when the same pass updates this file, the active guide checklist, validation notes, source map if needed, and `DEVELOP_DECISIONS.md`.
- A feature row can be `Partial` because another guide created reusable substrate. That does not by itself start or complete the owning numbered guide; the guide table and that guide's requirement checklist remain authoritative for guide-level status.
