# Develop Source Map

Last updated: 2026-06-22

This map describes the current Develop node implementation. Treat `docs/engineering/develop/spec_sources/DEVELOP_NODE_CONTEXT.txt` as the current-behavior source of truth, and verify code before changing behavior.

Composite text note, 2026-06-20:

- `src/Renderer/Internal/RenderPipelineImageGeneratorPass.cpp` now owns text-generator backdrop blur/padding composition in addition to text rasterization. Text generators can serialize `textBackdropBlur`, `textBackdropOpacity`, and `textBackdropPadding`, with `colorB` used as the backdrop tint.
- `src/Editor/Internal/EditorModuleComposite.cpp` and `src/Editor/Internal/EditorModuleRendering.cpp` now distinguish scalable raster requests from square scene-base sizing. Text generator composites still request scalable rasters, but their scene transform uses the actual uploaded/requested text raster aspect instead of a forced 256x256 square base.
- `src/Editor/NodeGraph/EditorNodeGraph.cpp` and `src/Renderer/Internal/RenderPipelineGraphExecution.cpp` now make Mix reference-canvas selection prefer non-generated image/RAW/custom sources over generated text/scalable sources when both inputs are connected. This keeps composite preview/export canvas dimensions stable for image-plus-text mixes regardless of whether the text generator is connected to Mix input A or B.

Validation organization note, 2026-06-20:

- `src/App/Validation/Suites/DevelopAutoSolveValidation.cpp` is now the small solver-only Auto-solve validation wrapper for `ValidateDevelopAutoSolveBehavior`.
- The former single giant scenario body is split into included scenario fragments under `src/App/Validation/Suites/DevelopAutoSolveValidationScenarios/`: `01_initial_solve_and_memory.inl`, `02_regional_subject_scenarios.inl`, `03_candidate_payload_and_scheduler.inl`, `04_rendered_feedback_scenarios.inl`, `05_core_dynamic_range_profiles.inl`, and `06_result_aggregation_and_report.inl`.
- The split is organization-only. Scenario order, shared local validation state, assertion expressions, failure-report content, CLI entrypoint, and helper ownership remain unchanged.
